Любое общение в сети происходит с помощью сетевых протоколов.

Сетевой протокол - это протокол передачи данных.

Есть два протокала которые работают ниже чем остальные.

## `OSI`

1. `Physical`
Передача данных начинается с физического уровня.
Например по LAN кабелю.

2. Потом идет `Data Link`, он связывает Физический уровень с Network.
Например Ethernet.

3. `Network` - это формирование ваших пакетов, самый главный передатчик.
Низкоуровневый протокол, то есть IP. Тут пакеты формируются и с одного IP адреса,
идут в другой IP адресс.

4. `Transport` - это или `TCP` или `UDP`

`TCP` - это протокол передачи данных, который ожидает ответ от получателя, получил ли
он данные. Если не получил, он отправит данные через некоторое время заново и сообщит отправителю об ошибке.

`UDP` - это тоже протокол передачи данных, но он не ожидает ответ от получателя,
а просто отправляет данные. Он быстрее чем TCP.

5. `Session` - подключаемая модель, то есть подключения к конкретному сокету.
То есть пакет уже дошел до получателя и ему выделяется сокет для принятия.
Не достаточно просто отправить пакет, этот пакет должен к какому-то порту придти.
Сокет - это IP + Порт.

6. `Presentation` - это уже видимый для нас уровень. Например картинка отправили
картинку нам и мы его приняли и уже его видем.

7. `Application` - это уровень с которым работают программисты. Это уже
сами протоколы которые мы сами настраиваем, маршрутизация и тд.
Например HTTP.

```csharp
	Socket listener = new Socket(AddressFamile.InterNetwork, SocketType.Stream, ProtocolType.Tcp);
```

Класс `Socket` это уже информация о самой сети, типе сокета и протоколе передачи данных.
Это класс для работы с сокетом.

`AddressFamile.InterNetwork` это значит IPv4

`TCP` общается по потоку, а `UDP` по датаграмма, 1 датаграмма = 64 кб.
