Класс `Parallel` предоставляет возможности для параллельной
обработки коллекций данных, выполнения итераций по массивам
и других структурам данных в параллельных потоках.

Не по теме.
`Lock` - только потоками внутри одного процесса 
`Mutex` - может управлять потоками внешних процессов
то есть процессами нескольких процессов
`Semaphore` `SemaphoreSlim` также как и `Mutex`

# Parallel For

Это обычный `For` но итерации выполняются несколькими потоками.
То есть допустим вам нужно 10 раз вывести ваше имя, он может выделить 
на это 5 потоков и выполнить весь цикл в две итерации.

Пример: 
Если вамм нужно сделать какое-то действие n раз
но при этом не важно в каком порядке, то так будет быстрее.
Но если порядок важен, то это не подойдет, потому что потоки выполняются
конкурентно. Допустим у вас есть массив из 10 имен и вы его выводите 
с помощью `ParallelFor` то он выведет быстрее, но не по порядку.

Он принимает три параметра
`fromInclusive` - Начальный индекс, включительно.
`toExclusive` - Конечный индекс, не включительно.
`body` - Делегат, который вызывается один раз за итерацию.
`state` - объкет класса ParallelLoopState

Свойства `ParallelLoopState`
`isExceptional`
Это свойство указывает, было ли выброшено исключение
во время выполнения одной из итераций цикла. Если в ходе выполнения
цикла произошло исключение, isExceptional будет равно true, и дополнительная
обработка исключения может потребоваться.

`LowestBreakIteration`
Это свойство указывает на наименьшую
итерацию, на которой была вызвана операция прерывания (break).
Если цикл был прерван при помощи оператора break, то
LowestBreakIteration содержит номер итерации, на которой
это произошло.

`Completed`
Это свойство указывает, завершилось ли
выполнение параллельного цикла. Если все итерации
завершились успешно или было выброшено исключение,
Completed будет равно true.

`IsStopped`
Это свойство указывает, было ли принудительно
прервано выполнение цикла (например, при помощи метода
ParallelLoopState.Stop()). Если цикл был остановлен,
IsStopped будет равно true, и выполнение цикла будет
завершено как можно скорее.

`ShouldExitCurrentIteration`
Этот метод указывает, должна
ли текущая итерация быть завершена досрочно. Это полезно,
если вы хотите завершить выполнение текущей итерации без
прерывания всего цикла.

`Interlocked` - это класс с помощью которого можно из нескольких 
потоков работать с одной переменной.

# Parallel ForEach

То же самое что и `Parallel For` 
В параметры он принимает элемент или колекцию из элементов
и выполняет делегат с каждым из элементов, также паралельно поделив итерацию на разные потоки.
``` csharp
var numbers = Enumerable.Range(1, 10);

Parallel.ForEach(numbers, number =>
{
    Console.WriteLine($"Processing number: {number}");
});
```
Если он выделит 5 потоков на выполнения этого  foreach то он выполнится за одну итерацию.
Это в 5 раз быстрее чем обычный foreach
